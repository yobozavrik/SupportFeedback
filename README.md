# Support Feedback

Цей репозиторій містить форму зворотного зв'язку для мережі "Галя Балувана" та бекенд, який приймає завантаження зображень і перенаправляє заявки у n8n.

## Швидкий старт

1. Встановіть залежності:
   ```bash
   npm install
   ```
2. Скопіюйте `.env.example` у `.env` та заповніть облікові дані сервісного акаунта Google.
3. Запустіть сервер:
   ```bash
   npm start
   ```

Сервер за замовчуванням слухає порт `3000` і надає ендпоінт `POST /upload/google-drive` для завантаження фотографій скарг.

## Налаштування сервісного акаунта Google Drive

1. Створіть проєкт у [Google Cloud Console](https://console.cloud.google.com/) і увімкніть для нього **Google Drive API**.
2. Створіть сервісний акаунт, згенеруйте для нього JSON-ключ та випишіть:
   - `client_email` → значення для змінної `GOOGLE_CLIENT_EMAIL`;
   - `private_key` → значення для змінної `GOOGLE_PRIVATE_KEY`. Ключ обов'язково потрібно обгорнути в лапки та замінити символи переносу рядка на `\n`, як показано у `.env.example`.
3. У Google Drive створіть або оберіть папку, до якої потрібно завантажувати файли, та поділіться нею з адресою сервісного акаунта (`client_email`) з правами **Editor**. Отримайте `folderId` із URL цієї папки та збережіть його у `GOOGLE_DRIVE_FOLDER_ID`.
4. За потреби вкажіть власний порт у змінній `PORT`.

Після коректного заповнення `.env` сервер автоматично ініціалізує клієнт Google Drive через `google.auth.JWT` і робить завантажені файли доступними за публічними посиланнями `webViewLink` та `webContentLink`.

## Завантаження фотографій із фронтенда

Фронтенд відправляє вибране користувачем зображення на `POST /upload/google-drive` у форматі `multipart/form-data` (поле `file`). Успішна відповідь містить `webViewLink` та `webContentLink`, один із яких використовується як `imageUrl` у заявці, що надсилається у n8n. Якщо завантаження не вдалось, користувач отримує попередження, а заявка відправляється без фотографії.
